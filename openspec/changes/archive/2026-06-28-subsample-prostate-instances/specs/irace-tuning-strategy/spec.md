# Delta for irace-tuning-strategy

## MODIFIED Requirements

### Requirement: Tiny PROSTATE training instances

Each irace setup (`irace_prostate/`, `irace_vns/`, `irace_tabu/`) SHALL train on
`PROSTATE_tiny_S*` instances rather than `MEDIUM_test` or full `PROSTATE`. Each setup's
`instances.txt` SHALL list at least 3 distinct `PROSTATE_tiny_S*` instances generated by
`scripts/subsample_instance.py` with different voxel seeds. Instance paths in
`instances.txt` SHALL NOT point at `MEDIUM_test_*` or full `PROSTATE_sampled*` files.

| Criterion | Value |
|-----------|-------|
| Min instances per setup | 3 |
| Source pool | `instances/PROSTATE_tiny_S*` (3 distinct seeds) |
| Forbidden entries | any path matching `MEDIUM_test_*` or `PROSTATE_sampled*` |

(Previously: trained on at least 2 `MEDIUM_test_K*` instances.)

#### Scenario: All 3 setups reconfigured to tiny instances
- GIVEN the `instances.txt` files reference `MEDIUM_test_K*` instances
- WHEN the reconfiguration is applied
- THEN every setup's `instances.txt` lists at least 3 `PROSTATE_tiny_S*` entries and zero `MEDIUM_test_*`/`PROSTATE_sampled*` entries

#### Scenario: Insufficient variants
- GIVEN only 2 `PROSTATE_tiny_S*` instances exist
- WHEN the setup is validated
- THEN it is rejected with a message stating at least 3 distinct tiny instances are required for irace training diversity

### Requirement: Reduced targetRunnerTimeout

All 3 scenario files SHALL set `targetRunnerTimeout = 60`. Tiny instances complete in
under 20s, so 60s gives adequate headroom for irace tuning while keeping the wall-clock
budget tractable.

(Previously: 120s for `MEDIUM_test` runs; originally 1500s for `PROSTATE`.)

#### Scenario: Timeout reduced to 60s
- GIVEN `scenario.irace` files contain `targetRunnerTimeout = 120`
- WHEN the change is applied
- THEN all 3 scenario files contain `targetRunnerTimeout = 60` and no occurrence of `120` or `1500` remains in that key

#### Scenario: Run exceeds 60s
- GIVEN a `PROSTATE_tiny_S*` irace run
- WHEN a single configuration iteration takes longer than 60s
- THEN irace marks the run as timed out (no crash) and continues to the next configuration
